# Microservices Contracts

This file defines practical REST and event contracts for the first extraction pass. Keep contract versions explicit and backward compatible.

## Common HTTP Headers

| Header | Required | Description |
| --- | --- | --- |
| `authorization` | external protected APIs | Bearer JWT from Auth Service |
| `x-correlation-id` | yes | Generated by Gateway if absent |
| `x-user-id` | internal protected APIs | Injected by Gateway after JWT validation |
| `x-user-role` | internal protected APIs | Injected by Gateway after JWT validation |
| `content-type` | write APIs | `application/json` |

## Error Shape

```json
{
  "success": false,
  "error": {
    "code": "ATTENDANCE_DUPLICATE",
    "message": "Attendance already marked for this student and date.",
    "details": {}
  },
  "correlationId": "uuid"
}
```

## Recognition Scan

`POST /recognition/scan`

Request:

```json
{
  "image": "data:image/jpeg;base64,...",
  "cameraId": "web-dashboard",
  "requestedBy": "user-id"
}
```

Response:

```json
{
  "success": true,
  "recognized": true,
  "recognition": {
    "studentId": "student-id",
    "faceLabel": "raj_patel",
    "confidence": 0.91,
    "modelVersion": "2026.06.25-001",
    "box": { "x": 12, "y": 20, "width": 140, "height": 140 },
    "quality": { "score": 0.86, "issues": [] }
  }
}
```

## Mark Attendance

`POST /attendance/mark-from-recognition`

Request:

```json
{
  "studentId": "student-id",
  "faceLabel": "raj_patel",
  "recognition": {
    "confidence": 0.91,
    "modelVersion": "2026.06.25-001",
    "eventId": "recognition-event-id"
  },
  "markedAt": "2026-06-25T08:55:00.000Z",
  "location": {
    "latitude": 23.0225,
    "longitude": 72.5714,
    "accuracy": 25
  }
}
```

Response:

```json
{
  "success": true,
  "duplicate": false,
  "record": {
    "attendanceId": "attendance-id",
    "studentId": "student-id",
    "dateKey": "2026-06-25",
    "status": "Present",
    "markedAt": "2026-06-25T08:55:00.000Z"
  }
}
```

## Event Topics

RabbitMQ exchange option:

- Exchange: `attendance-system.events`
- Type: `topic`
- Routing keys use event type, for example `attendance.marked`.

Kafka option:

- Topic per domain: `student.events`, `face.events`, `attendance.events`, `system-events.events`.
- Event type remains inside the envelope.

  "eventVersion": 1,
